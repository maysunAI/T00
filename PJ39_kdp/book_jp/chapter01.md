# 第1章：なぜClaudeはすぐに「忘れて」しまうのか？

## 1.1 あなたは一人ではない

Claude Codeを使い始めて、最初にぶつかる壁があります。

「昨日あれだけ詳しく説明したのに、今日また最初から説明しなければならない」

「どこかに答えを書いてくれたはずなのに、会話が長くなると見つけられない」

「別のプロジェクトで同じ質問をしたら、また同じことをゼロから答えてくれた」

これはあなたが特別なのではありません。AIとの会話における最も普遍的な悩みです。

## 1.2 Claudeのメモリーとはどういうものか

Claude（およびほとんどのAI）はセッション間で記憶を保持しません。

技術的に言えば：

- **1回の会話（コンテキストウィンドウ）** = Claudeが「今」覚えていること
- **会話を閉じる** = すべてがリセット
- **新しい会話を開く** = 白紙の状態からスタート

これは欠陥ではなく、設計です。プライバシーとコスト管理のためです。

しかし、開発者にとっては大きな問題です。

### 具体例：何が起きているのか

```
月曜日：
あなた「このプロジェクトの認証システムはJWT+Redisで構築しています。
        セッション有効期限は24時間で、リフレッシュトークンは7日です」
Claude「わかりました。では修正を…」（詳細な回答）

火曜日（新しいセッション）：
あなた「昨日の認証の続きをやりたい」
Claude「申し訳ありませんが、昨日の会話の内容は持っていません。
        認証システムの詳細を教えていただけますか？」
```

毎朝、同じ説明を繰り返す。これが現実です。

## 1.3 問題を数字で見る

私の経験では、1回のセッションの最初の**10〜15分**は以下に費やされます：

| 作業 | 時間 | 本当に必要か |
|------|------|-------------|
| プロジェクトの背景説明 | 3分 | ❌ 前回も説明した |
| 前回どこまでやったか確認 | 4分 | ❌ ファイルを見ればわかる |
| AIが理解しているか確認 | 3分 | ❌ ルールを書けば不要 |
| 本当の作業 | 残り | ✅ これだけしたい |

週5日、毎日15分 × 5日 = **週75分の無駄**。

月に換算すると約5時間。年間だと60時間以上、AIへの「おはよう、また僕です」に費やしています。

## 1.4 AI00フレームワークの誕生

私はこの問題を解決するために、シンプルな方法を考えました。

**「AIが忘れるなら、忘れないファイルに書けばいい」**

これがAI00フレームワークの核心です。

複雑なシステムは不要です。必要なのは3つのMarkdownファイルだけ：

```
temp01_req.md    ← あなたが要求を書く場所
temp02_草稿.md   ← 下書き・メモ
temp03_reply.md  ← AIが回答を書く場所（行番号付き）
```

そして、1つのルールファイル：

```
CLAUDE.md        ← AIへの永続的な指示書
```

## 1.5 どう解決するのか：3つの仕組み

### 仕組み①：行番号通信プロトコル

従来の方法：
```
あなた「昨日話した認証の件なんですが、あのredisの設定を…」
Claude「どの設定でしょうか？詳しく教えてください」
```

AI00の方法：
```
あなた「req 45行後」
Claude（45行目を読む）「SESSION_REDIS_TTL=86400 の件ですね。了解しました」
```

**ポイント**：「45行後」という3文字で、前の会話の文脈を完全に復元できます。

### 仕組み②：RESUME.mdによるセッション引き継ぎ

会話を終える前に「再見（さようなら）」と言うだけ：

```
あなた「再見」
Claude → RESUME.md を自動更新：
  「前回：認証システムのredisタイムアウト修正完了
   次回：フロントエンドのトークン更新ロジックから」
```

次の会話：
```
あなた「RESUME.mdを読んで続きをやって」
Claude「前回はredisタイムアウトを修正しました。今日はトークン更新から始めますね」
```

### 仕組み③：CLAUDE.mdによるルール永続化

一度書いたルールは永遠に有効：

```markdown
# CLAUDE.md
- 常に日本語で回答する
- コードの変更は必ずdiffで表示する
- テストを書かずにコードを変更しない
```

新しいセッションを開いても、これらのルールは自動的に読み込まれます。

## 1.7 まず試してみましょう

読むだけでなく、今すぐ試せます。

**所要時間：3分**

```
1. temp01_req.md を作成
2. 中に書く：「1. 自己紹介してください」
3. Claudeに言う：「req 1行後」
4. Claudeが temp03_reply.md に回答する
5. 行番号（📌）を確認する
```

これだけです。もう文脈は消えません。
