# 第2章：10分でセットアップ — 最初の「再見」まで

> **この章で学ぶこと**：AI00フレームワークをゼロから構築し、「再見」コマンドで自動アーカイブするまでの全手順を体験します。

---

## 2.1 必要なもの（所要時間：2分）

まず、以下が揃っているか確認してください。

| ツール | バージョン | 確認方法 |
|--------|-----------|---------|
| Claude Code | 最新版 | `claude --version` |
| VS Code（推奨） | 任意 | — |
| 作業フォルダ | 任意の場所 | — |

> **Claude Codeがない場合**：[claude.ai/code](https://claude.ai/code) からインストール

---

## 2.2 フォルダ構造を作る（所要時間：2分）

ターミナルで以下を実行します：

```bash
# 作業フォルダを作成（例：T00）
mkdir T00
cd T00

# 3つのテンプファイルを作成
echo "# temp01_req.md — 要求ファイル" > temp01_req.md
echo "# temp02_草稿.md — メモ" > "temp02_草稿.md"
echo "# temp03_reply.md — AIの回答" > temp03_reply.md
```

完成後のフォルダ構造：

```
T00/
├── temp01_req.md    ← あなたが要求を書く場所
├── temp02_草稿.md   ← メモ・下書き
└── temp03_reply.md  ← AIが回答を書く場所
```

---

## 2.3 CLAUDE.mdを作る（所要時間：3分）

`CLAUDE.md` はAIへの「永続的な指示書」です。以下の内容で作成します。

```bash
# T00フォルダで実行
touch CLAUDE.md
```

`CLAUDE.md` を開き、以下を貼り付けます：

```markdown
# 私のAI00設定

## 基本ルール
- 常に日本語で回答する
- 実質的な回答はすべて `temp03_reply.md` に書く
- CLIには短い確認（例：「Reply 更新（第XX行）」）のみ出力
- 回答の末尾に `📌 req 第X-X行 → reply 第XX行 ✅` を付ける

## req/replyシステム
- ユーザーが「req XX行後」と言ったら → temp01_req.md のXX行目以降を読む
- 回答は temp03_reply.md に追記する（上書きしない）

## セッション終了
- ユーザーが「再見」と言ったら → RESUME.md を自動更新する
```

> **ポイント**：このファイルはClaude Codeを開くたびに自動で読み込まれます。一度書けば永遠に有効です。

---

## 2.4 最初のreqを書く（所要時間：1分）

`temp01_req.md` を開き、以下を追記します：

```
今日やること：
1. このプロジェクトの概要を説明してください
2. req/replyシステムのテストをしてください
```

保存します。

---

## 2.5 Claude Codeを起動する（所要時間：1分）

T00フォルダでClaude Codeを起動します：

```bash
claude
```

Claude Codeが起動したら、以下と入力します：

```
req 3行後
```

Claudeが `temp01_req.md` の3行目以降を読み、`temp03_reply.md` に回答を書きます。

### 確認ポイント

`temp03_reply.md` を開くと、以下のような構造で回答が追記されているはずです：

```
══════════════════════════════
❓ [2026-06-11 10:00]
原文：
1. このプロジェクトの概要を説明してください
2. req/replyシステムのテストをしてください
───────────────────────────────
回答

1. プロジェクト概要
...（Claudeの回答）

📌 req 第3-5行 → reply 第1行 ✅
```

これが動いていれば、AI00フレームワークは正常に機能しています。

---

## 2.6 VS Codeで分割表示する（推奨）

VS Codeを使っている場合、画面を分割すると効率が大幅に上がります。

```
┌──────────────────┬──────────────────┐
│  temp01_req.md   │ temp03_reply.md  │
│  （要求を書く）  │ （回答が来る）   │
└──────────────────┴──────────────────┘
```

**設定方法**：
1. `temp01_req.md` を開く
2. `Ctrl+\`（Windowsの場合）でパネルを分割
3. 右側に `temp03_reply.md` を開く

これで、左に要求を書きながら、右でAIの回答をリアルタイムで確認できます。

---

## 2.7 最初の「再見」を実行する

これがAI00の最大の特徴です。

Claude Codeに入力します：

```
再見
```

Claudeが自動的に以下を実行します：

```
✅ RESUME.md 更新中...
✅ SESSION_LOG.md に記録中...
```

### RESUME.mdの中身を確認

```bash
cat RESUME.md
```

以下のような内容が自動生成されているはずです：

```markdown
# RESUME.md — AI引き継ぎファイル

## 前回日付：2026-06-11

## やったこと
- AI00フレームワークのセットアップ完了
- req/replyシステムの動作確認
- temp03_reply.md に回答テスト済み

## 次のステップ
1. CLAUDE.mdにプロジェクト固有のルールを追加
2. 本格的な開発タスクを開始
```

---

## 2.8 次のセッションで確認する

別の日にClaude Codeを起動したら、以下を入力するだけです：

```
RESUME.mdを読んで続きをやって
```

Claudeが前回の状態から即座に再開します。説明の繰り返しはゼロです。
