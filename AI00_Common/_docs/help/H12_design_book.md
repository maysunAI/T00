# H12 — 设计书工作流

> 给人看。如何用设计书让 AI 改程序。有实例。`[AI00]`

---

## 什么是设计书

设计书 = 你和 AI 都认可的"蓝图"。AI 按蓝图改代码，人在蓝图上写意见。

---

## 两种模式

| 模式 | 文件名格式 | 谁用 |
|------|---------|------|
| 省token版（默认）| `D01_xxx.md` | AI读，含 `[批注]` 区 |
| 易懂版 | `Design_xxx.md` + `Design_xxx_edit.md` | 人读，人在edit版写意见 |

切换方法：在 `MODE.md` 写：
```
current_mode: token_saving   ← 省token版
current_mode: human_friendly ← 易懂版
```

---

## 快速开始（省token模式，3步）

**第1步：让AI生成设计书草稿**

在 req 里写：
```markdown
## 需求
帮我为 PJ20 的登录功能写设计书（D01_login.md）
```

AI 会在项目目录下生成 `D01_login.md`。

**第2步：你在 `[批注]` 区写修改意见**

打开 `D01_login.md`，找到 `[批注]` 标记，在它下面写：
```
[批注]
登录失败次数限制改为3次，不是5次。
```

**第3步：告诉 AI 批注位置**

```
D01_login.md 第25行批注有修改，请按批注改代码。
```

AI 读批注 → 改代码 → 清空批注区。

---

## 设计书文件位置

放在各项目目录下：
```
F:\T00\PJ20_myapp\
├── CLAUDE.md
├── D01_login.md       ← 设计书（省token版）
├── D02_dashboard.md
└── src\
    └── ...
```

---

## 和 req/reply 的关系

| 用什么 | 适合场景 |
|--------|---------|
| req | 小需求、对话式需求 |
| 设计书 | 大功能、需要留存蓝图的需求 |
| 两者结合 | 在 req 里说「按 D01 第3章改」|

---

## 完整模板

```markdown
# D01 — 功能名称

## 概述
（AI起草，1-3句话）

## 数据结构
（字段列表）

## 业务逻辑
（步骤列表）

## 界面说明
（组件描述）

[批注]
（人写修改意见，AI处理后清空）
```
