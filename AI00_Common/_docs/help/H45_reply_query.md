# H45 — 如何查找 Reply 里的信息

> **问题**：reply 里有很多有用的内容，我想再看某条，怎么说？

---

## 方法一：说行号（最直接）

```
你在 reply 第 442-448 行写的内容，再说一遍
```

```
reply 第 531 行是什么
```

AI 会直接读取那几行并重新解释给你。

---

## 方法二：说关键词

```
你之前说过 GitHub Topics 怎么设置，再说一次
```

```
上次那个 Reddit 发帖步骤在哪
```

AI 会去 reply 里搜索相关内容。

---

## 方法三：说时间/话题

```
今天（或昨天）我们讨论过 PJ32 的事，再说一遍
```

```
关于 skill pool 的讨论在哪里
```

---

## 方法四：直接在 VSCode 里看

1. 按 `Ctrl+P` → 输入 `temp03_reply` → 打开文件
2. 按 `Ctrl+G` → 输入行号（如 `531`）→ 回车 → 直接跳转
3. 按 `Ctrl+F` → 输入关键词搜索

---

## 常用信息对照表

每次 AI 回答后，会在末尾写：
```
📌 req 第X-X行 → reply 第XX行 ✅
```

这个行号就是这次回答的起始位置，收藏起来随时可查。
