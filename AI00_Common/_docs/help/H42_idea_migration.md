# H42 — 把一个想法套用到新市场

> 适用场景：你在 A 地区/平台看到一个成功模式，想在 B 地区/平台复制

---

## 核心认知

> **成功模式不能照搬，只能「转译」。**

直接复制 = 90% 失败  
理解底层逻辑再适配 = 成功率大幅提升

---

## IDEA 迁移 3 步法

### 第 1 步：解构原模式

问自己：**「这个模式里，到底是什么在起作用？」**

| 问题 | 说明 |
|------|------|
| 核心价值是什么？ | 省钱？省时间？娱乐？情感？ |
| 谁在用？为什么用？ | 用户画像、使用动机 |
| 为什么在这里成功？ | 政策、平台、文化、时机 |
| 如果这些条件消失，还能用吗？ | 找出核心，去掉不可移植的部分 |

**例：AI 写歌（中国市场）**
- 核心价值：极低成本生产大量音乐内容
- 用户：短视频创作者、自媒体
- 为什么在中国成功：抖音流量经济 + 音乐版权意识较弱 + AI 工具便宜
- 核心 = 「低成本内容生产」，不核心 = 「抖音平台」

---

### 第 2 步：目标市场调研

**4 个维度的差异分析：**

| 维度 | 要看什么 | 工具 |
|------|---------|------|
| 平台生态 | 主流视频/音乐平台是什么？算法偏好？| 直接用目标平台 |
| 版权环境 | 版权执法严格吗？AI 音乐的法律地位？| 查当地法律 |
| 文化偏好 | 音乐审美、语言、节奏偏好 | 听当地热门内容 |
| 消费能力 | 付费意愿、竞品定价 | AppStore/Google Play 价格 |

**例：中国 AI 写歌 → 日本**

| 维度 | 中国 | 日本 | 差异处理 |
|------|------|------|---------|
| 平台 | 抖音 | YouTube Shorts / TikTok JP | 类似，可迁移 |
| 版权 | 执法较松 | 非常严格（JASRAC）| 必须用无版权音乐或原创 |
| 文化 | 流行/国风 | J-Pop / 动漫风 | 需专门训练/提示词 |
| 付费 | 低价竞争 | 愿意为质量付费 | 定价可更高 |

---

### 第 3 步：差异化适配（不是照搬，是转化）

找出「可迁移」和「需要改造」的部分：

```
✅ 可直接用：技术工具（AI 模型）、产品概念、营销思路
⚠️ 需适配：语言/文化内容、平台运营方式、定价策略
❌ 不能搬：依赖特定政策/平台的部分
```

**适配清单（以 AI 写歌→日本为例）：**
- [ ] 确认 AI 生成音乐在日本的版权现状
- [ ] 收集日本热门 BGM 风格（动漫、Lo-fi、J-Pop）
- [ ] 调整 prompt 使生成风格更符合日本审美
- [ ] 研究 YouTube Shorts 的算法偏好（vs 抖音）
- [ ] 日语界面 / 日本用户客服能力
- [ ] 定价参考：日本音乐市场的现有服务定价

## 其他行业的迁移例子

| 原模式 | 原市场 | 迁移目标 | 关键适配点 |
|--------|--------|---------|-----------|
| AI 陪伴应用 | 中国 | 台湾/东南亚 | 繁体中文 + 文化语境 |
| 短视频带货 | 中国 | 东南亚 | 当地网红生态 + 货币 |
| AI 写作助手 | 英文市场 | 日文市场 | 日语语法特殊性 |
| 订阅制工具 | 美国 | 日本 | 日本偏好买断制 |

---

## 常见失误

| 失误 | 原因 | 正确做法 |
|------|------|---------|
| 照搬中文 UI 直接推 | 懒 | 最少做本地化翻译 |
| 只看用户数量，不看付费意愿 | 流量思维 | 研究变现路径 |
| 忽略当地法规 | 不了解 | 先查版权/隐私法 |
| 对标中国竞争强度 | 惯性 | 当地竞争可能更少 |
