## 旧版 LLM 和现在的区别

### 旧版（2022年以前）
```
LLM = 只用训练数据回答
→ 训练截止日期之后的信息不知道
→ 「现在股价多少？」→ 无法回答
```

### 现在（2024年以后）
```
LLM + Web 搜索工具 = 可以访问实时信息
→ 「现在股价多少？」→ 用 Google 搜索后回答
→ 「e+ 现在有票吗？」→ 确认网站后回答
```

---

## Gemini 的情况

Gemini 与 **Google 搜索的整合**是最强的。

| 功能名 | 说明 |
|--------|------|
| Google Search Grounding | 可在 Gemini API 中启用的官方功能 |
| 效果 | 实时查询最新新闻、价格、库存 |
| 费用 | 包含在 API 调用费中（无额外费用）|
| 使用方式 | PJ90 调用 Gemini 时通过选项启用 |

---

## PJ90 中的应用

```
Firebase 定时执行
→ 调用 Gemini（开启 Google Search）
→ 「e+ 上 ○○ 的票现在有吗？」
→ Gemini 搜索 Google → 实时确认
→ 「有」→ 给用户推送通知
→ 「没有」→ 等下次检查
```

因此 PJ90 可以：
- 不需要自己写网页爬虫
- 直接借用 Gemini 的搜索能力
- 网站结构改变也能灵活应对

---

## 总结

| | 旧版 LLM | 现在的 Gemini |
|--|---------|------------|
| 训练数据截止后的信息 | ❌ 不知道 | ✅ 可以搜索 |
| 实时价格 | ❌ | ✅ |
| 票务库存确认 | ❌ | ✅ |
| 新闻 | ❌ | ✅ |

PJ90 的监控通知功能，就是依靠 Gemini 的 Web 搜索能力来实现的。

---

## 对 PJ90 的影响

- **用户聊天时**：同一对话内搜索结果共享，效率高
- **PJ90 定时监控任务**：每次都是新的API调用，每次都重新搜索，保证实时性
- **费用**：搜索次数 = API调用次数，但用户自己的 Gemini 免费额度足够个人使用
