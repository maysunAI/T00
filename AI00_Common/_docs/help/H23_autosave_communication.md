# H23 — 自动保存规则 + 高效交流方式

> 来源：用户提问（2026-06-07）

---

## 什么内容 AI 会自动保存？什么需要问？

| 情况 | AI 动作 | 理由 |
|------|--------|------|
| 项目功能想法（目标明确）| 直接保存到项目 IDEAS.md，告知位置 | 位置确定，内容有价值 |
| 用户说「写入」「保存到」| 立即执行，不问 | 用户已指定 |
| 内容可归属多个地方 | 先问「保存到 A 还是 B？」| 避免放错位置 |
| 影响架构/规则的决策 | 先问确认，再保存 | 重要，需用户确认 |
| 临时讨论中的想法（未确认）| 不保存，只列在📦建议里 | 用户可能还要改 |
| 调试输出 / 错误信息 | 不保存 | 临时性内容 |

**简单记忆**：有明确项目 + 明确位置 → 直接保存；不确定 → 先问；临时 → 不保存

---

## 📦 建议保存机制

每条 reply 末尾，AI 会列出：
```
📦 建议保存：
- [内容简述] → [建议文件路径]
```
用户说「保存」或「全部保存」时 AI 执行。

## 让 AI 发挥最好的交流方式

### 询问类 vs 指令类

| 你的表达 | 适合场景 | AI 做什么 |
|---------|---------|---------|
| 「您能告诉我…」「有哪些…」| 想了解信息、比较选项 | 分析后给推荐 + 理由 |
| 「请帮我做…」「直接做…」| 已决定方向，要执行 | 直接执行，不再讨论 |

两种都正确，看目的选。

### 让 AI 输出最精准的公式

```
目标 + 背景 + 限制
```

**例子**：
- ❌ 「做一个登录」
- ✅ 「帮我做登录功能（目标），参考 D01 第2章设计（背景），不要改 main.py（限制）」

| 你说 | AI 理解 | 输出质量 |
|------|--------|---------|
| 「推荐图表库」| 不知道你的技术栈 | 通用回答 |
| 「我用 React + Python，推荐图表库」| 清楚你的场景 | 精准推荐 |

### 快速反馈词（参见 H17）

| 你说 | AI 做什么 |
|------|---------|
| `good` | 保存为正向 feedback，继续这个方向 |
| `perfect` | 强化信号，完全保持 |
| `not good` | 立刻调整 + 保存为避免记录 |

## CLI vs req — 什么时候用哪个？

| 场景 | 推荐 | 理由 |
|------|------|------|
| 脑暴、探索想法 | CLI（直接打字）| 即时互动，快速迭代 |
| 快速问一个问题 | CLI | 比写文件快 |
| 正式需求确认 | req | 有记录，AI 引用行号精确 |
| 长段文字/代码 | req | CLI 不方便写长文 |
| 多步骤复杂任务 | req | 可以分段，防止遗漏 |
| 粘贴错误/命令输出 | req | 便于 AI 完整分析 |

**把 CLI 中有效信息写入 reply**：
- 讨论完直接说：「这个结论保存到 reply」
- 或说：「这段好，保存到 [文件路径]」
- 或在 req 里：「上面 CLI 讨论的结果是 X，请记录」

不用自己整理，告诉 AI 记就行。
