# H11 — 对人的要求（AI00 合作标准）

> 给人看。人达到这个水平，和 AI 合作效率最高。
> 不符合？没关系，下面有学习路径。

---

## 基础要求（必须）

| 要求 | 说明 | 怎么学 |
|------|------|--------|
| 会保存文件 | Ctrl+S，在告诉 AI 之前保存 | VSCode 基础 |
| 会数行号 | VSCode 左边有行号，点击跳转 | 看一遍就会 |
| 会复制粘贴 | 把 AI 给的命令贴到终端 | 基础操作 |
| 安装了 Claude Code | `npm install -g @anthropic-ai/claude-code` | H10 第一步 |

## 进阶要求（提高效率）

| 要求 | 说明 | 怎么学 |
|------|------|--------|
| 了解 CLAUDE.md 作用 | 规则写在这里，AI 每次自动读 | 读 H06 |
| 会用行号告知格式 | 「req 50行后」这样说话 | 读 H01 |
| 会写结构化需求 | `## 需求 1. xxx 2. xxx` | 读 H03 |
| 知道 req/reply 工作流 | 写需求→告知行号→看回答 | 读 H02 |

## 高级要求（密切合作）

| 要求 | 说明 |
|------|------|
| 理解 FUTURE_FEATURES.md | 「以后想做」不立即实现，记录在这里 |
| 会看 tasks/TASK_INDEX.md | 了解哪些做了哪些没做 |
| 会用再见协议 | 说「再见」触发存档 |
| 会用 /clear 恢复 | 说「请读取 RESUME.md」 |

---

## 学习路径（从零开始）

```
第1天：H10（快速开始）→ 做一个 hello world
第2天：H01 + H02（行号通信 + req/reply 工作流）
第3天：H03（格式写法）→ 写一个真实需求试试
第1周：H08（快速参考）← 这个要背下来
随时查：H00（索引）
```

---

## 如果你不符合要求

**AI 会帮助你：**
- 如果文件未保存 → AI 提醒你先保存
- 如果格式不清楚 → AI 会问清楚再做
- 如果需求太模糊 → AI 会给你示例格式
- 如果找不到文件 → AI 告诉你路径

**你需要做的：**
- 告诉 AI 你不确定的地方
- 不要怕说错，AI 会引导你

---

## 人和 AI 的分工

| 人做 | AI做 |
|------|------|
| 决定做什么（方向）| 决定怎么做（实现）|
| 写需求（req）| 写代码/文档 |
| 做最终判断 | 提供选项和建议 |
| 负责创意 | 负责执行 |
| 保持好奇心 | 提供知识和启发 |

> 人是创造力的源头，AI 是放大器。你的想法越清晰，AI 帮你做的就越好。
