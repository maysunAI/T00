# H03 — 让 AI 更清楚的写法

> 给人看。AI 平时不读此文件。

---

## 推荐格式

用 `##` 标类型，用数字或 `-` 列内容，背景单独写。

### 提问
```
## 问题
1. useState 和 useRef 什么时候用哪个？
2. 为什么 useEffect 触发了两次？
```

### 提需求
```
## 需求
1. 登录按钮改成蓝色
2. 加「记住我」复选框

## 背景
用户反映登录页太单调
```

### 确认 / 反馈
```
## 确认
- 方案 A 可以，按这个做
- 第3条不需要，跳过
```

---

## ❌ 不清晰的写法

| 写法 | 问题 |
|------|------|
| 「那个按钮」 | 哪个？AI 会猜错 |
| 「之前说的功能」 | 跨 session 后 AI 不记得 |
| 多件事混在一段话 | AI 可能漏掉其中一件 |
| 没有类型标签 | AI 不知道是提问还是需求 |

---

## 规则总结

- `##` 明确类型：问题 / 需求 / 背景 / 确认
- 多项用数字编号
- 避免代词「那个」→ 换成具体文件名或功能名
- 背景和需求分开写
- 一个 `##` 块只讲一件事
