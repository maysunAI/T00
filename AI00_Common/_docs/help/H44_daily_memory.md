# H44 — 每天记住做了什么

> 问题：和 AI 聊了几十轮，回头完全不记得做了什么、决定了什么

---

## T00 已经在帮你记

你不需要手动记录。T00 有两个自动机制：

| 文件 | 谁读 | 何时写入 | 内容 |
|------|------|---------|------|
| `RESUME.md` | AI 读 | 每次「再见」后自动写 | 上次做了什么、下一步是什么 |
| `SESSION_LOG.md` | 人读 | 每次「再见」后自动追加 | 详细记录（日期+做了什么+未完成）|

**关键**：会话结束时说「**再见**」，AI 会自动写入这两个文件。

---

## 查今天（或上次）做了什么

### 方法1：看 RESUME.md（最快）

```
RESUME.md 在 T00 根目录
```

第一行就是「上次日期」和「做了什么（3-5条）」，30 秒读完。

### 方法2：看 SESSION_LOG.md（最详细）

```
路径：AI00_Common/_docs/discussion/SESSION_LOG.md
```

按日期区块排列，每次会话一个区块：
```
## 2026-06-11
### 做了什么
- 写了 H42/H43/H44
- 讨论了论坛引流计划
### 未完成
- GitHub About 描述还需手动改
```

### 方法3：Grep 搜索关键词

想找「上次关于 PJ17 做了什么决定」：
```
在 temp03_reply.md 搜索：PJ17
在 SESSION_LOG.md 搜索：PJ17
```

在 Claude Code 里说：
```
搜一下 reply 里关于 PJ17 的内容
```

## 内容太多看不完——每周复盘

每周花 10 分钟做一次复盘：

1. 打开 `SESSION_LOG.md`
2. 看本周所有区块的「做了什么」
3. 问 AI：「帮我总结本周进展，并列出下周优先事项」

---

## 「我完全不知道上次停在哪里」

发生这种情况时，会话开始时说：
```
请读取 RESUME.md 继续上次的工作。
```

AI 会读取 RESUME.md，告诉你上次做到哪里，并问你从哪里继续。

---

## 常见问题

**Q：RESUME.md 和 SESSION_LOG.md 有什么区别？**

| | RESUME.md | SESSION_LOG.md |
|-|-----------|----------------|
| 读者 | AI（下次会话时自动读）| 你自己 |
| 长度 | 精简（1屏）| 完整（累积增长）|
| 更新方式 | 每次「再见」**覆写**（只保留最新）| 每次「再见」**追加**（全部保留）|

**Q：我没有说「再见」就关掉了，怎么办？**

SESSION_LOG 没有写入。但 `temp03_reply.md` 里还有本次所有回答，下次会话时说：
```
req 1 以后（或「请看 reply 最后几行」）
```
手动触发 AI 写入 SESSION_LOG。

**Q：reply 行数太多了，怎么知道今天的回答在哪里？**

Grep 搜索今天日期：
```
搜索 reply 里的「2026-06-11」
```
会找到今天所有回答的 ❓ 标记行。

---

## 提升记忆效果的习惯

| 习惯 | 做法 |
|------|------|
| 结束时说「再见」| 触发自动存档，最重要 |
| 重要决定加 📦 | 决定时加保存建议，不只在 reply 里 |
| 项目进展写到项目文件夹 | 不要只留在 reply |
| 每周看一次 SESSION_LOG | 避免方向漂移 |
