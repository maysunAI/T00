# R09 — 需求预估与完成度评估规则

> 适用：AI00框架下所有项目

---

## 开始前：需求可达性预估

用户说「**开始做 [功能/项目]**」时，AI 先做预估：

**预估格式：**
```
需求可达性评估：
- 核心功能：✅ 可实现（约X小时）
- 功能A：✅ 可实现
- 功能B：⚠️ 需确认（依赖第三方 API，可能有限制）
- 功能C：❌ 当前技术栈难实现

预估完成度：85%
```

**规则：**
- 预估 ≥ 90%：直接开始
- 预估 70-89%：简单说明限制，问用户是否继续
- 预估 < 70%：**先讨论，不直接做**，给出替代方案

---

## 完成后：需求完成度报告

功能完成时，AI 附上完成度报告：

**报告格式：**
```
需求完成报告：
✅ 功能A — 完成
✅ 功能B — 完成
⚠️ 功能C — 部分完成（说明原因）
❌ 功能D — 未完成（说明原因）

总完成度：X / Y 项（XX%）
```

---

## 触发词

| 用户说 | AI 做 |
|--------|-------|
| 「开始做」「帮我实现」| 先做预估，≥90% 再动手 |
| 「直接做，不用评估」| 跳过预估直接做 |
| 「完成了吗」「做完了吗」| 输出完成度报告 |

---

## 与 R08 的配合

R08（先搜索）→ R09（预估）→ 实现 → R09（评估报告）

完整顺序：
1. 搜索现有方案（R08）
2. 预估需求可达性（R09）
3. 用户确认 → 开始实现
4. 完成后输出报告（R09）
