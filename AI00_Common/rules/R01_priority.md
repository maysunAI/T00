# R01 — 规则优先级系统

> 状态：v1.0（2026-06-10）
> 目的：明确所有规则的优先顺序，冲突时以高级别为准。

---

## 优先级总览

```
L1 — Claude Code 效率（最高优先）
  ↓
L2 — Reply 优先（沟通质量）
  ↓
L3 — T00 框架（工作流规范）
  ↓
L4 — 开发规范（质量保障）
```

冲突时：**高级别规则优先，低级别规则让步**。

---

## Level 1 — Claude Code 效率（不可违反）

| # | 规则 | 来源 |
|---|------|------|
| 1.1 | 自动加载总量 ≤ 5k tokens（≈ 333 行） | CLAUDE.md 跳过规则 |
| 1.2 | 项目专用规则放项目 CLAUDE.md，禁止放全局 `.claude/rules/` | 本文 |
| 1.3 | 大型功能用 skills 按需加载，不堆进 CLAUDE.md | 本文 |
| 1.4 | 跳过无关文件夹（Z01_archive / Z02_personal / PJzz / 历史项目） | CLAUDE.md |
| 1.5 | req 更新时先验证文件行数，行数不足提示「文件可能未保存」 | CLAUDE.md 沟通规则 |

---

## Level 2 — Reply 优先（不可省略）

| # | 规则 | 来源 |
|---|------|------|
| 2.1 | 所有实质性回答必须写进 `temp03_reply.md` | CLAUDE.md 最重要规则 |
| 2.2 | CLI 只输出简短确认（如「Reply 已更新（第XX行）」），不重复 reply 内容 | CLAUDE.md（2026-06-10更新） |
| 2.3 | Reply 格式：原文逐字 + AI整理 + 答案 + 行号 + **日期 HH:MM**（含时间，不只写日期）| CLAUDE.md |
| 2.4 | 每条回答末尾注明：`📌 req 第X-X行 → reply 第XX行 ✅` | CLAUDE.md |
| 2.5 | 重要内容会话结束前必须保存到对应文件（📦 建议格式） | CLAUDE.md |
| **2.6** | **Reply 中写"下一步/待执行"必须同步执行**，禁止留"下次再做" | H37（2026-06-10）|

## Level 3 — T00 框架（工作流规范）

| # | 规则 | 来源 |
|---|------|------|
| 3.1 | 项目内容存项目文件夹，不散落 T00 根目录 | CLAUDE.md 存放规则 |
| 3.2 | 说「再见」触发：SESSION_LOG + RESUME + 下次命令 | CLAUDE.md 再见协议 |

## Level 4 — 开发规范（有条件执行）

| # | 规则 | 来源 |
|---|------|------|
| 4.1 | trust: high → 修改前说明原因；trust: low → 可大胆重写 | R06 |
| 4.2 | 重大变更前先 commit（新建多文件/修改规则/删除文件） | CLAUDE.md Git规则 |
| 4.3 | 2000行备份（由 PostToolUse Hook 自动检测，不需AI主动检查）| CLAUDE.md |
| 4.4 | 自动安装工具（npm/pip/MCP）：**仅用户明确遇到缺失问题时触发** | CLAUDE.md Auto-Install（降级） |
| 4.5 | 小任务直接做；大任务（5+文件）派子Agent | R05 |
| 4.6 | 敏感文件不提交（.env/.pem/.key），PreToolUse Hook 拦截 | settings.local.json |
