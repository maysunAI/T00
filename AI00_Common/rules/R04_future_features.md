# R04 — 未来功能管理规则

**目的**：记录「将来想做的事」，同时不影响当前开发流程。

---

## 触发关键词

用户使用以下关键词时 → 不立即实现，记录到 FT 文件：

- 「以后」「未来」「将来」「有空的时候」「想法」
- 「考虑中」「暂时先不做」「有机会的话」

---

## Claude 的操作流程

1. 检测到关键词 → 追加到 `docs/REQUESTS/FUTURE_FEATURES.md`
2. **不开始实现**
3. 告知用户：「已保存到未来计划，需要实现时请告知」

---

## 转为实现的条件

用户明确表达以下意图时才开始实现：
- 「开始做○○」
- 「帮我实现○○」
- 「现在要做○○」

---

## FUTURE_FEATURES.md 记录格式

```markdown
## [YYYY-MM-DD] — [功能概要]

**详细**：[想做什么]
**目标页面**：[在哪里添加]
**期望行为**：[具体行为]
**优先级**：低 / 中 / 高
**状态**：未启动 / 考虑中 / 进行中 / 完成
```
