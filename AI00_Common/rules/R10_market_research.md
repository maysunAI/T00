# R10 — 项目立项前市场调研规则

> 适用：AI00框架下所有新项目立项时
> 触发：用户说「建项目」/ 「开始PJxx」时自动执行（大项目必做，小项目可省）

---

## 触发条件

| 项目规模 | 判断 | 是否必做调研 |
|---------|------|------------|
| 大项目（预计2小时以上）| 新功能/新领域 | ✅ 必做 |
| 中项目（1-2小时）| 有竞品风险 | ⚠️ 建议做 |
| 小项目/改动 | 明确无竞品 | ❌ 可跳过 |

用户说「不用调研，直接做」→ 跳过。

---

## 调研内容（固定模板）

### 1. 同行调查（至少10个）

| # | 产品名 | 平台 | Stars/下载量 | 技术栈 | 特点 |
|---|--------|------|-------------|--------|------|
| 1 | ... | ... | ... | ... | ... |
| ... | | | | | |

搜索关键词模板：
```
[项目类型] app github 2024 2025
awesome [关键词] list
[功能描述] open source
[项目类型] site:producthunt.com
```

### 2. 竞争力分析

- **我们的差异化**：和同行相比，我们做什么他们没做的？
- **我们的弱点**：哪些地方比不过成熟产品？

### 3. 失败原因（同行中失败的项目）

- 为什么失败？（产品问题 / 市场时机 / 技术 / 运营）
- 我们如何避免同样的错误？

### 4. 痛感分析

- 解决什么问题？
- 这个问题有多痛？（用户每天遇到？偶尔？）
- 现有解决方案有多差？

### 5. 成功原因（同行中成功的项目）

- 成功的核心要素？
- 我们能复制哪些成功要素？

### 6. 护城河

- 什么让用户留下来不走？（数据积累 / 网络效应 / 高切换成本 / 独特技术）
- 我们的护城河是什么？

### 7. Mapping 分析（现在 → 未来 → 建议）

| 维度 | 内容 |
|------|------|
| **现在**（当前市场格局）| 竞品现状，我们的差距和优势 |
| **未来**（1-3年展望）| 市场趋势，AI对该领域的影响 |
| **建议**（行动方向）| 推荐切入点，差异化策略，MVP范围 |

### 8. AI 补充分析

- 有没有我没想到的风险？
- 有没有更好的切入角度？
- 市场规模估算（用户数 × 付费率 × 客单价）
- 推荐的 MVP 功能集合（最小可行产品）

### 8. AI 总结建议

```
推荐做 / 谨慎做 / 不建议做
原因：[1-3条]
建议切入点：[具体方向]
预估完成度：XX%（参考R09）
```
