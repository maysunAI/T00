# R03 — 用户手动修改保护规则

**问题**：Claude 编辑文件时，可能覆盖用户之前手动修改的内容。

---

## 用户端的操作方式（3种）

### 方式1：在代码注释中标注（推荐）

```vue
<!-- [手动修改] 按钮已移到右侧。请勿还原 -->
```

```python
# [手动修改] 此处理顺序是有意为之，请勿修改
```

```typescript
// [手动修改] 此类型转换是 API 限制所致
```

### 方式2：在会话开始时告知

```
「[文件名] 的 [功能名] 我手动修改过了，请保留」
```

### 方式3：在 CLAUDE.md 的「手动修改记录」表中填写

直接在 `CLAUDE.md` 末尾的手动修改记录表中追加记录。

---

## Claude 的义务

1. 看到 `[手动修改]` 注释时，该部分绝对不修改
2. 编辑常用文件前，先查看手动修改记录
3. 用户告知「手动修改过」时，立即记录到 CLAUDE.md

---

## 手动修改记录表格式（放在 CLAUDE.md 中）

```markdown
| 文件 | 功能/位置 | 修改内容 | 记录日 |
|------|---------|---------|-------|
| `frontend/src/views/ImportView.vue` | 导入标签页—按钮 | 移到右侧 | 2026-05-25 |
```
