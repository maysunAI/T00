# AI006 — Rules 文件使用技巧

> rules/ 文件夹是 CLAUDE.md 的详细延伸。本文说明如何读懂、修改、新增规则。

---

## 1. 规则体系结构

```
CLAUDE.md                        ← 主规则（每次对话自动读取）
    │
    └── rules/                   ← 详细规则（CLAUDE.md 引用）
        ├── R01_code_rules.md    ← 编码标准
        ├── R02_session_rules.md ← 会话结束规则
        ├── R03_manual_edit_protection.md ← 保护用户修改
        └── R04_future_features.md ← 未来功能处理
```

**关系**：CLAUDE.md 是摘要，rules/ 是全文。Claude 会在需要时自动读取对应的规则文件。

---

## 2. 各规则文件说明

### R01_code_rules.md — 编码规范
**控制**：Claude 写代码时用什么风格

| 内容 | 默认值 |
|-----|-------|
| Python 风格 | PEP8 + 类型提示 + Pydantic |
| Vue/TypeScript | Vue3 Composition API + strict 模式 |
| 注释风格 | 写"为什么"，不写"做什么" |
| 变更范围 | 只改需求内的代码 |

**常见自定义**：
```
# 追加到 R01 末尾
- 变量名用英文，注释用中文
- 函数长度不超过 50 行
- 不使用 any 类型
```

---

### R02_session_rules.md — 会话结束规则
**控制**：每次对话结束，Claude 自动创建哪些文件

| 文件类型 | 路径 | 用途 |
|---------|-----|-----|
| D 文件 | `docs/P01_changes/` | 本次变更的文件清单 |
| QA 文件 | `docs/QA01_sessions/` | 会话内容存档 |
| BG 文件 | `docs/BG01_bugs/` | Bug 解决记录 |
| CL 日志 | `docs/CL01_changelog/` | 变更日志追加 |

> 也可以直接输入 `/session-end` 手动触发。

---

### R03_manual_edit_protection.md — 手动修改保护
**控制**：防止 Claude 覆盖你手动修改的内容

**三种保护方式**：

| 方式 | 操作 | 适用场景 |
|-----|-----|--------|
| 代码注释 | 在代码里写 `# [手动修改]` | 具体某行/某块代码 |
| 会话告知 | 对话开始时说「我修改了 XXX，不要动它」 | 当次对话 |
| CLAUDE.md 记录表 | 填入文件/位置/内容 | 长期保护 |

**CLAUDE.md 记录表用法**（最可靠）：
```markdown
## 用户手动修改记录

| 文件 | 功能/位置 | 修改内容 | 记录日 |
|------|---------|---------|-------|
| frontend/App.vue | 顶部导航 | 手动调整了颜色 | 2026-05-31 |
```

---

### R04_future_features.md — 未来功能管理
**控制**：说"以后想做 XXX"时，Claude 只存档不实现

**触发关键词**：「以后」「未来」「将来」「有空再做」「想法」

**存档位置**：`docs/REQUESTS/FUTURE_FEATURES.md`

**实现条件**：你说「开始做○○」或「帮我实现○○」时才执行。

---

## 3. 如何添加自定义规则

### 方法 A：直接追加到对应 rules 文件（推荐）
```
打开 rules/R01_code_rules.md
在末尾追加你的规则
```

### 方法 B：让 Claude 写规则
```
"以后[场景]，你要[行为]。把这条规则添加到 rules/R01_code_rules.md。"
```

### 方法 C：新建规则文件
```
在 rules/ 创建 R05_xxx.md
在 CLAUDE.md 的参考文档表中加一行引用
```

---

## 4. 规则优先级

```
当次对话的口头说明   ← 最高优先（本次对话有效）
         ↓
CLAUDE.md 规则       ← 每次对话有效
         ↓
rules/ 文件          ← 详细规则（CLAUDE.md 引用时生效）
```

> 如果临时想让 Claude 不遵守某条规则：直接在对话里说「这次不用遵守 XXX 规则」。

---

## 5. 常用自定义规则示例

```markdown
# 追加到 CLAUDE.md 末尾或对应 rules 文件

## 我的自定义规则

- 每次修改代码后，先运行测试再报告完成
- 生成 API 文档时用中文说明
- 数据库字段名用下划线命名，不用驼峰
- 输出文字说明时限制在 200 字以内
```
