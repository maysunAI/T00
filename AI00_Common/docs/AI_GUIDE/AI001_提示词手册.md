# AI001 — 提示词手册（和 Claude 对话的关键 Prompt）

> 本文件是和 Claude AI 对话时的参考手册。
> 不同场景使用不同的说话方式，可以大幅减少来回沟通次数。

---

## 1. 功能开发类

### 1.1 新增功能
```
在「[页面名]」页面，添加[功能描述]。
效果：[期望的用户体验]
```
示例：`在「复习」页面，添加一个「暂停」按钮。效果：点击后进度暂停，再次点击继续。`

---

### 1.2 修改现有功能
```
把「[页面名]」的「[功能/组件]」改成[新的样式或行为]。
原因：[为什么要改]
```

---

### 1.3 修复 Bug
```
在「[页面名]」，[操作步骤]时，出现了[异常现象]。
期望：应该是[正确行为]。
```

---

### 1.4 批量修改（多个需求）
```
## [页面名]
1. [需求1]
2. [需求2]

## [页面名2]
1. [需求1]
```
> 用编号 + 页面分组，Claude 会按顺序执行，不会遗漏。

---

## 2. 理解代码类

### 2.1 解释功能实现
```
帮我解释一下「[功能]」是怎么实现的。不需要很深，大概说流程就好。
```

### 2.2 查找问题原因
```
为什么[现象]？帮我找原因。
```

### 2.3 确认功能是否正确
```
帮我确认一下「[功能]」目前的实现是否正确。
预期行为：[描述]
```

---

## 3. 想法类（还没想好怎么做）

```
[随意描述想法，写多写少都行]
→ 写入 INBOX 文件后，告诉 Claude："INBOX 有新内容"
```
Claude 会提出 2-3 个方案，你选择后再实现。

---

## 4. 规划未来功能（不立即执行）

```
以后我想[功能描述]。用在[场景]。
```
> Claude 会自动将此内容保存到 `docs/REQUESTS/FUTURE_FEATURES.md`，不会立即开始开发。

---

## 5. 规则设定类（写进 CLAUDE.md 永久生效）

```
以后每次[场景]，你都要[行为]。把这个写进规则。
```
示例：`以后每次我说「以后想做X」，你都要把它存到 FUTURE_FEATURES 文件。把这个写进规则。`

---

## 6. 程序启动/调试类

### 6.1 启动程序
```
帮我启动程序。
```

### 6.2 程序报错
```
程序报错了：
[粘贴错误信息]
帮我找原因并修复。
```

---

## 7. AI 记忆类

### 7.1 让 AI 记住某个信息
```
记住：[重要信息]。以后对话都参考这个。
```

### 7.2 让 AI 忘记某个信息
```
忘记：[信息]。以后不要再用这个。
```

### 7.3 确认 AI 记住了什么
```
你现在记得哪些关于我的信息？列出来。
```

## 8. 快速对话模板（速查表）

| 场景 | 推荐说法 |
|------|---------|
| 想做新功能 | 「在[页面]添加[功能]，效果是[期望]」 |
| 功能不对 | 「[页面]里[操作]时，出现[现象]，应该[期望]」 |
| 看不懂代码 | 「解释一下[功能]的实现，大概流程就好」 |
| 有模糊想法 | 写入 INBOX → 「INBOX 有新内容」 |
| 未来想法 | 「以后我想[功能]」（自动存档）|
| 规则设定 | 「以后每次[场景]，你要[行为]，写进规则」 |
| 启动程序 | 「帮我启动程序」 |
| 确认变更 | 「帮我确认一下今天改了什么」 |

---

## 9. 注意事项

1. **不需要说"请"** — 直接说要做的事，Claude 会礼貌回应
2. **不需要解释技术** — 说用户行为和期望结果就好
3. **用中文** — 整个项目使用中文对话
4. **未来 vs 现在** — 说「以后想做」就存档；说「帮我做」就立即执行
5. **分批** — 一次说 5 个需求就够，太多容易遗漏
