# AI007 — Memory 系统使用技巧

> Memory 是 Claude 的跨对话记忆。写进去的东西，下次新对话自动记得。

---

## 1. Memory 文件在哪里

```
C:\Users\[你的用户名]\.claude\projects\[项目路径的转义]\memory\
    ├── MEMORY.md           ← 索引（每次对话自动加载）
    ├── user_*.md           ← 你的角色/习惯
    ├── feedback_*.md       ← 行为约定
    ├── project_*.md        ← 项目进度/决策
    └── reference_*.md      ← 外部工具/文档位置
```

**查看方法**：在对话框输入 `/memory`

---

## 2. 四种记忆类型

### `user` — 关于你自己
记录你的角色、技术水平、工作习惯，让 Claude 调整解释方式。

```
"记住：我是 Python 后端开发，对前端不熟悉，解释前端时要更详细。"
```

---

### `feedback` — 行为约定
告诉 Claude 什么该做/不该做，长期生效。

```
"记住：不要在回答末尾总结刚才做了什么，我自己能看到。"
"记住：每次修改数据库前必须先备份，这个要写进记忆。"
```

> 这是最有用的类型。把让 Claude 改掉的习惯都写进去，省得每次重复说。

---

### `project` — 项目信息
记录项目进度、重要决策、特殊背景。

```
"记住：用户认证模块用的是 JWT，不是 Session，以后涉及认证都参考这个。"
"记住：这个项目的数据库不能用 DELETE，只能软删除（is_deleted 字段）。"
```

---

### `reference` — 外部位置
记录哪里能找到什么信息。

```
"记住：Bug 记录在 Notion 的「项目日志」页面。"
"记住：设计稿在 Figma，链接在 REQUESTS/FUTURE_FEATURES.md 里。"
```

---

## 3. 常用操作

### 让 Claude 记住某件事
```
"把这个记到记忆里：[内容]"
"记住：[内容]。以后对话都参考这个。"
```

### 查看 Claude 记得什么
```
"/memory"          ← 在对话框输入
"你现在记住了哪些关于我的信息？列出来。"
```

### 让 Claude 忘记某件事
```
"忘记：[信息]，从记忆里删掉。"
"把记忆里关于 XXX 的内容删掉。"
```

### 更新记忆
```
"更新记忆：我已经从初学者变成中级了，以后不需要解释基础概念。"
```

---

## 4. 最值得写进记忆的内容

| 类型 | 例子 |
|-----|-----|
| 你的技术水平 | 「精通 Python，Vue 刚开始学」 |
| 你讨厌的行为 | 「不要在回答里用表情符号」 |
| 项目特殊约定 | 「所有 API 返回格式固定为 {code, data, msg}」 |
| 长期禁止事项 | 「不要修改 [手动修改] 标注的代码」 |
| 常用工具位置 | 「设计稿在 Figma，链接是 XXX」 |

---

## 5. Memory vs CLAUDE.md 的区别

| | Memory | CLAUDE.md |
|---|--------|----------|
| 作用范围 | 所有项目通用 | 当前项目专用 |
| 适合写什么 | 你个人的习惯和偏好 | 项目的具体规则 |
| 位置 | `~/.claude/projects/.../memory/` | 项目根目录 |
| 修改方式 | 对话里说「记住/忘记」 | 直接编辑文件 |

> **原则**：关于"你这个人"的信息 → Memory；关于"这个项目"的规则 → CLAUDE.md

---

## 6. 注意事项

- Memory 文件是纯文本，可以直接用记事本打开修改
- MEMORY.md 是索引，内容超过 200 行会被截断，保持简洁
- 记忆可能过时，重要信息建议定期确认一下
- 不需要记代码、文件内容、git 历史——这些直接读文件更准确
