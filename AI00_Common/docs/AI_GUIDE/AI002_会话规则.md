# AI002 — 会话规则（每次对话 Claude 必须执行的事项）

> 本文件是 `CLAUDE.md` 的补充说明，详细解释每条规则的触发条件和执行方式。

---

## 规则总表

| # | 触发条件 | 必须执行的操作 |
|---|---------|--------------|
| **R-A** | **修改任何功能前** | **先读设计书 → 先改设计书 → 再改代码** |
| **R-B** | **每次对话结束** | **QA 末尾附用户原文，每条标注对应修改编号** |
| R01 | 修改了代码 | 更新设计书对应章节 |
| R02 | 每次对话结束 | 创建修改文件记录（D文件）|
| R03 | 每次对话结束 | 创建 QA 问答记录 |
| R04 | 发生报错/异常 | 记录到 Bug 文件 |
| R05 | 用户说「以后/未来/想法」| 存档到未来计划，不立即执行 |
| R06 | 每次对话结束 | 告知用户创建了哪些文件 |
| R07 | 修改代码后 | 告知用户修改的文件名和内容 |

---

## 各规则详细说明

### R01 — 设计书同步更新

**触发**：任何代码文件被修改

**执行内容**：
- 找到设计书中对应的章节
- 更新该章节描述，反映最新实现
- 更新文档顶部的「最終更新」日期

---

### R02 — 修改文件记录（D 文件）

**触发**：对话结束时，本次有代码或文档变更

**命名格式**：`D[NN]_修改文件[YYYYMMDD]_[当日序号]([简述]).md`

**内容格式**：
```
# D[NN] — 修改文件记录
日期：YYYY-MM-DD　描述：[简述]

## 修改内容
### 1. [模块名] — [变更说明]
文件：[路径]
变更：[具体改了什么]

## 修改文件一览
| 文件 | 变更类型 | 说明 |
```

---

### R03 — QA 问答记录

**触发**：每次对话结束

**命名格式**：`QA0XX_session_YYYY_MM_DD.md`（同一天多次：`_2`, `_3`...）

**必须包含**：用户原始提问文本附在末尾，每条标注对应了哪个解答。

---

### R04 — Bug 记录

**触发**：发现报错、启动失败、功能异常

**时机**：解决后立即记录

**内容格式**：
```
## [Bug 名称]
- 现象：[报错信息或异常表现]
- 原因：[根本原因]
- 解决方法：[具体步骤或代码修改]
- 修改文件：[路径 + 行号]
```

---

### R05 — 未来计划存档

**触发词**：用户说了「以后」「未来」「想法」「计划」「还没实现」等词

**执行内容**：
1. 保存到 `docs/REQUESTS/FUTURE_FEATURES.md`
2. 告知用户「已保存到未来计划，不立即执行」
3. **不开始任何代码修改**

---

### R06 — 告知用户创建的文件

**触发**：对话结束时

**格式**：使用 Markdown 链接，方便点击
```
## 本次创建/修改的文件
- [QA文件](相对路径)
- [D文件](相对路径)
```

---

### R07 — 代码修改通知

每次修改代码文件后简短说明：修改了哪个文件 + 大概改了什么。

---

## 文件编号规则

三位数编号（001, 002...），同类型编号连续递增。

| 前缀 | 类型 | 示例 |
|------|------|------|
| D | 修改记录 | D15_修改文件... |
| QA | 问答记录 | QA028_session... |
| BG | Bug记录 | BG002_问题记录... |
| FT | 未来计划 | FT002_... |
| DS | 设计书 | DS003_... |

---

## 常见错误

| 错误行为 | 正确行为 |
|---------|---------|
| 改了代码，没有更新设计书 | 改完代码后立即更新设计书对应章节 |
| 没有创建 D 文件 | 对话结束前必须创建 |
| 没有创建 QA 文件 | 对话结束前必须创建 |
| 用户说"以后想做X"就开始写代码 | 存档到 FUTURE_FEATURES，不执行 |
| 文件链接用纯文本路径 | 必须用 Markdown 链接格式 |
