# AI004 — Claude Code 操作技巧

> 让 Claude Code 用起来更顺手的实用技巧集合。

---

## 1. Slash 命令（对话框直接输入）

| 命令 | 说明 | 推荐场景 |
|------|------|---------|
| `/clear` | 清空对话历史，保持记忆 | 话题切换、对话过长时 |
| `/compact` | 压缩对话历史（保留摘要）| 节省 Token，长对话时 |
| `/cost` | 显示本次对话 Token 用量和费用 | 查费用 |
| `/plan` | 进入计划模式（只规划不执行）| 讨论方案，需要确认再动手 |
| `/fast` | 切换 Fast 模式（Opus 快速输出）| 大量代码生成时 |
| `/memory` | 查看/管理记忆文件 | 确认 Claude 记得什么 |
| `/help` | 查看帮助 | 忘记命令时 |

## 2. Memory 系统 — 跨对话记忆

Claude Code 在以下路径保存持久记忆，**每次新对话自动加载**：

```
C:\Users\[用户名]\.claude\projects\[项目路径]\memory\
├── MEMORY.md           ← 索引（自动加载）
├── user_profile.md     ← 你的角色、习惯
├── project_*.md        ← 项目概述
├── feedback_*.md       ← 行为约定
└── reference_*.md      ← 外部资源位置
```

**主动让 Claude 记住**：
```
"请把这个记到记忆里：[内容]"
```

| 记忆类型 | 用途 |
|---------|------|
| `user` | 你的角色、技术水平、习惯 |
| `feedback` | 行为约定（"不要这样"/"以后都这样"）|
| `project` | 项目进度、决策背景 |
| `reference` | 外部工具/文档的位置 |

## 3. CLAUDE.md — 每次对话自动读取的规则书

**位置**：项目根目录 `CLAUDE.md`

Claude Code 启动时自动读取，相当于给每次对话的「开场说明」。

**设定永久规则的方法**：
```
"以后每次[场景]，你都要[行为]。把这个写进 CLAUDE.md 规则。"
```

---

## 4. 新对话开始时（最高效的方式）

```
1. 不需要重新介绍项目背景（记忆系统和 CLAUDE.md 会自动加载）
2. 直接说要做什么：
   "继续[项目名]，今天要做：[1-3行说明]"
3. 跨项目时说：
   "切换到[项目名]，要做：[说明]"
```

## 5. 结构化需求描述（减少来回确认）

**功能需求**：
```
页面：[具体页面]
需求：[一句话]
期望行为：
- [具体1]
- [具体2]
```

```
现象：[描述]

2.
```

## 8. 终端快捷操作

**`! 命令` — 在对话框直接运行 shell 命令**：
```
! git status          ← 查看 git 状态
! dir F:\T00          ← 列出文件夹内容
! python main.py      ← 运行脚本
! gh auth login       ← GitHub 认证
```
结果直接出现在对话里，无需切换终端窗口。非常适合快速确认状态。

**代码块标记**：

| 标记 | 语言 | 含义 |
|------|------|------|
| ` ``` ` | Markdown | 代码块（不是注释）|
| `'''` | Python | 多行字符串（常被用作注释，但本质是字符串）|
| `//` | JavaScript | 单行注释 |
| `#` | Python/Shell | 单行注释 |

## 9. 费用控制技巧

| 技巧 | 效果 |
|------|------|
| 对话过长时用 `/compact` | 压缩历史，继续节省 |
| 新话题用 `/clear` | 清空历史，从干净状态开始 |
| 大量代码操作时开 `/fast` | Opus 快速模式，效率更高 |
| 设置月上限 | 防止超额 → https://console.anthropic.com/settings/limits |
| 问题描述精准 | 减少来回确认的轮次 |
| Bug 报告附 F12 错误信息 | 排查时间从 10 分钟缩短到 1 分钟 |
