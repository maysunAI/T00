---
name: t00-estimate
description: 需求预估 + 完成度报告（R09）。开始实现前评估可达性；完成后输出报告。
---

用户说「/t00-estimate $ARGUMENTS」时，根据 $ARGUMENTS 决定模式：

## 模式1：开始前预估（$ARGUMENTS 含「开始」「实现」「做」）

输出格式：
```
需求可达性评估：
- 核心功能：✅ 可实现（约X小时）
- 功能A：✅ 可实现
- 功能B：⚠️ 需确认（依赖第三方 API，可能有限制）
- 功能C：❌ 当前技术栈难实现

预估完成度：85%
```

规则：
- ≥ 90%：直接开始实现
- 70–89%：说明限制，问用户是否继续
- < 70%：先讨论，给出替代方案，不直接做

## 模式2：完成后报告（$ARGUMENTS 含「完成了吗」「做完了」「报告」）

输出格式：
```
需求完成报告：
✅ 功能A — 完成
✅ 功能B — 完成
⚠️ 功能C — 部分完成（原因）
❌ 功能D — 未完成（原因）

总完成度：X / Y 项（XX%）
```

## 模式3：无参数

列出当前项目或当前任务的预估状态。
