请对以下项目范围执行市场调研任务。

## 参数说明

用户调用时可附加参数（若无参数则默认 active）：

| 参数 | 范围 |
|------|------|
| `active` | 当前活跃项目（PJ10+，排除 PJzz）|
| `all` | 全部项目（含 PJzz_future / PJzz_paused）|
| `future` | 仅 PJzz_future 未来想法项目 |
| `paused` | 仅 PJzz_paused 暂停项目 |
| `pjXX` | 指定单个项目（如 pj20）|
| `pjXX pjYY` | 指定多个项目 |

用户输入的参数：$ARGUMENTS

## 执行步骤

1. **确认范围**：根据参数列出要处理的项目清单，输出给用户确认
2. **检查已有调研**：有 MARKET_RESEARCH.md 的项目 → 只报告文件位置，不重复做
3. **并行研究**：没有调研的项目 → 启动并行 Agent 搜索 Top 10 竞品
4. **输出格式（R10标准）**：
   - Top 10 竞品表格
   - 差异化优势 vs 弱点
   - Mapping：现在 / 未来 / 建议
   - 可行性结论（推荐/谨慎/不建议 + 理由）
5. **设计书**：无 D01_design.md 的项目 → 自动生成草稿
6. **汇总报告**：所有项目的可行性对比表

## 注意

- PJzz_arch_old 是历史存档，访问前先提醒用户（增加 token 消耗）
- 已有调研文件的项目直接报文件路径，不重复搜索
- 写入各项目的 MARKET_RESEARCH.md
