> ⚡ 接近 context 上限（约 100k tokens）时提前触发本命令，生成 Handoff 摘要以便下次无缝衔接。

请执行会话结束存档流程。按以下顺序完成：

1. **确认本次变更** — 列出本次会话中修改或新增的所有文件

2. **创建 D 文件（变更记录）** → `Docs/P01_changes/D[NN]_[YYYYMMDD]_[概要].md`
   - 包含：变更文件列表、每个文件的具体改动内容

3. **创建 QA 文件（会话记录）** → `Docs/QA01_sessions/QA[NNN]_session_[YYYY_MM_DD].md`
   - 包含：主要作业内容、实现要点、用户原始提问（末尾必须附上）

4. **如有 Bug 修复** → `Docs/BG01_bugs/BG[NN]_[YYYYMMDD]_[概要].md`
   - 包含：现象、根本原因、解决方法、修改文件

5. **追加变更日志** → `Docs/CL01_changelog/CL001_log.md`
   - 格式：`## [YYYY-MM-DD] — [简述]`

6. **更新 RESUME.md** — 写入：今日完成项 / 下一步 / 快速上下文（3-5条）

7. **向用户报告** — 以 Markdown 链接格式列出所有新建文件

编号规则：
- D 文件：两位数（D01, D02...）查现有最大编号+1
- QA 文件：三位数（QA001, QA002...）查现有最大编号+1
- BG 文件：两位数（BG01, BG02...）
- 同一天多次会话：文件名末尾加 `_2`, `_3`
